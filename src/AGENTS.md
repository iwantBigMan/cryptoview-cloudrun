# AGENTS.md

## Communication
- 모든 설명, 작업 계획, 변경 요약, 테스트 결과는 한국어로 작성한다.
- 사용자가 영어로 요청하지 않는 한 한국어로 답변한다.
- 코드의 변수명, 함수명, 클래스명은 기존 프로젝트 스타일을 따른다.
- 코드 주석은 필요한 경우에만 작성하고, 기존 코드가 영어 주석이면 영어를 유지한다.
- 프롬프트는 실제 사용한 내용을 기준으로 작성하며, 의미를 유지한 상태에서 가독성을 고려해 문장을 재구성하여 기록한다.

## Work Style
- 작업 전 현재 브랜치와 변경 파일을 확인한다.
- `.idea` 폴더 파일은 사용자가 명시적으로 요청하지 않는 한 수정하지 않는다.
- 기능 구현 시 기존 아키텍처와 폴더 구조를 우선 유지한다.
- 수정 후 변경한 파일 목록과 핵심 변경 내용을 한국어로 요약한다.
- 빌드나 테스트를 실행했다면 결과를 한국어로 정리한다.
- 작업이 끝난 후에는 prompt.md에 사용한 프롬프트를 작성한다.
- prompt.md 작성 시 기능 관련 프롬프트, 테스트 관련 프롬프트, 기타 프롬프트로 구분하여 작성한다.
- 커밋 메시지는 제목을 포함한 형식으로 작성하며, 한국어를 기본 언어로 사용한다.
